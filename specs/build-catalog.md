# Spec: `build-catalog` — generated catalog and index pipeline

Status: ready for implementation. Depends on: nothing (build this first).
Consumers: `doctor` (D05/D07/D11, `--fix`), `sync` (regeneration step), all agent
workflows that currently regenerate `INDEX.md` by prompt-following.

## 1. Goal

Make frontmatter in `papers/*.md` the single authoritative metadata store and
derive everything else deterministically, in one direction:

```
papers/*.md (frontmatter + first line of ## Contribution)
        │  scripts/build-catalog.js
        ▼
   catalog.json          ← machine view (agents filter with jq/grep instead of
        │                   opening notes)
        ▼  same script, same run
   INDEX.md              ← human/agent browse view, grouped by topic
```

`INDEX.md` is no longer regenerated by prompt-following; workflows call this
script. This eliminates the drift class that `doctor` checks D05/D07/D11 exist to
catch (the repo has live drift today: 4 papers listed "Unassigned" in INDEX.md
while clustered in `topics/signature-transformers.md`, with empty `topics:`
frontmatter).

## 2. Stories

1. **Bootstrap `package.json`** — private package, `js-yaml` as the only
   dependency, `node --test` wired as the test script. Add `node_modules/` to
   `.gitignore` (already done if the repo-level gitignore update shipped).
2. **`scripts/build-catalog.js`** — full-scan generation of `catalog.json` +
   `INDEX.md` with write-if-changed.
3. **`--changed <path...>` incremental mode** — patch affected records only;
   output must be byte-identical to a full rebuild.
4. **Workflow/doc handover** — update `.claude/commands/*.md`,
   `.codex/prompts/*.md`, `CLAUDE.md`, and `AGENTS.md` so every workflow that
   previously said "regenerate INDEX.md" now says "run
   `node scripts/build-catalog.js`", and add the rule: *never hand-edit
   `catalog.json` or `INDEX.md`; run the script.*

## 3. CLI

```
node scripts/build-catalog.js [--changed <path>...] [--check] [--split-topics]
```

- default: full scan of `papers/*.md`, regenerate both outputs (write-if-changed).
- `--changed <path>...`: update only the records for the named note files, then
  re-emit outputs. Must produce byte-identical results to a full rebuild.
- `--check`: build in memory and exit 1 if the on-disk outputs differ (CI/doctor
  use); writes nothing.
- `--split-topics`: additionally emit `indexes/<topic>.md` (same line format) and
  shrink `INDEX.md` to a table of contents linking them. **Implemented but
  default-off**; the switch is flipped when the corpus passes ~200 papers.

Exit codes: 0 ok, 1 `--check` mismatch or unparseable note frontmatter (report
`file: message`, skip record, still emit others), 2 unexpected I/O failure.

## 4. `catalog.json` schema

One record per paper, sorted by citekey (stable diffs), schema versioned,
committed to git (agents on fresh clones need it; diffs double as a change log):

```json
{
  "schema": 1,
  "generated": "2026-07-07T00:00:00Z",
  "papers": [
    {
      "citekey": "tongSigFormerSignatureTransformers2023",
      "title": "SigFormer: Signature Transformers for Deep Hedging",
      "authors": ["Tong", "Nguyen-Tang", "Lee", "Tran", "Choi"],
      "year": 2023,
      "venue": "ICAIF 2023",
      "url": "https://arxiv.org/abs/2310.13369",
      "zotero": "",
      "depth": "sections",
      "topics": [],
      "added": "2026-07-06",
      "updated": "2026-07-06",
      "contribution": "one-line Contribution text, verbatim from the note",
      "path": "papers/tongSigFormerSignatureTransformers2023.md"
    }
  ]
}
```

Rules:

- `contribution` = first non-empty line under `## Contribution` (heading missing
  or section empty → `""`; multi-line paragraph → first line only).
- `generated` is the only timestamp; it must **not** force a write when nothing
  else changed (compare content excluding `generated`, or set `generated` only
  when records changed).
- Unknown extra frontmatter keys are ignored, not errors.

## 5. `INDEX.md` emission

Format is fixed by CLAUDE.md:

```
- [citekey](papers/<citekey>.md) — title | year | depth
```

- Grouped by topic (one `## <topic>` section per topic map name that appears in
  any `topics:` list), papers sorted year desc, then citekey asc for ties.
- A paper with N topics appears in N sections.
- Papers with empty `topics:` go under `## Unassigned`.
- Keep the two-line format explainer at the top of the file (as today).
- Em dash `—` as separator (current INDEX.md drifted to `--`; the rebuild fixes
  it).

## 6. Acceptance criteria

- [ ] On the current repo, emits `catalog.json` with 4 records including
      contribution lines, and an `INDEX.md` grouped correctly (all 4 papers under
      `## Unassigned` until their `topics:` frontmatter is fixed — the script
      reflects frontmatter, it does not read topic maps).
- [ ] Running it twice in a row → zero file changes, mtimes intact.
- [ ] `--changed papers/x.md` output byte-identical to a full rebuild
      (test-enforced).
- [ ] `--check` exits 1 after hand-corrupting `INDEX.md`, 0 after regeneration.
- [ ] Note with corrupt frontmatter: reported by path, other records still
      emitted, exit 1.
- [ ] All command prompts, CLAUDE.md, and AGENTS.md updated per Story 4.

## 7. Tests

- Frontmatter edge cases: quoted strings, missing optional fields, non-ISO dates,
  empty lists, unknown keys.
- Determinism: shuffle input file order → identical bytes.
- Contribution extraction: missing heading; empty section; multi-line paragraph
  (first line only); heading present but next heading immediately follows.
- `--split-topics` emits `indexes/<topic>.md` correctly and the TOC-form
  `INDEX.md`.
- Write-if-changed: no-op run leaves mtimes unchanged (compare stat before/after).

## 8. Out of scope

Embeddings/RAG (explicitly deferred); reading topic maps (they are human prose;
frontmatter is the only input); any Zotero interaction.
